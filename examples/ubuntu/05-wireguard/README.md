# 05 — WireGuard VPN

Builds on 04: sets up a WireGuard VPN hub on the server. The local machine connects as a client peer.

## What it does

1. All hardening from 01–04
2. Installs WireGuard
3. Configures the server as a VPN hub (`wg0` on `10.10.0.1/24`)
4. Auto-generates a client key pair and config
5. Saves WireGuard state locally for reuse on re-runs

## Key settings

- `wireguard.enabled: true`
- `wireguard.interface: wg0`
- `wireguard.address: 10.10.0.1/24` — server VPN address
- `wireguard.peer_address: 10.10.0.2/32` — client VPN address
- `wireguard.endpoint: 203.0.113.10:51820` — server's public endpoint

## Key generation

```bash
# Generate server key pair
wg genkey | tee .secrets/wg-server.key | wg pubkey > .secrets/wg-server.pub
```

The client key pair is auto-generated by nodeforge and persisted to `~/.wg/nodeforge/{host}/client.key`.

## Goss verification

The accompanying `05-wireguard.goss.yaml` validates the expected server state after apply.
